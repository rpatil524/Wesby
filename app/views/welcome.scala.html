Manual configuration

To manually configure your data portal you may modify the default configuration settings found in application.conf. You can find this file in the conf folder, inside the Wesby installation folder.

This file should should include some documentation comments to ease the process.

Customising your data portal

Wesby is fully customizable, allowing you to adapt your data portal by means of standard HTML and CSS templates.

If you just want to adapt the CSS styles to your site, you can override the default styles by adding your custom styles to the main.css file, inside the public/ folder.

By default, Wesby uses the Foundation Framework to simplify the portal building process and provide a responsive interface. You can choose not to use this framework by modifying the stylesheets in the main.scala.html template.

In the following sections you can find more information and help about the customization process, Wesby's anatomy, and how to customize the HTML templates.

Templates

Wesby is built on top of Play Framework, which uses the Twirl template engine to provide a customisable user interface. This template engine binds the resources data to the HTML templates using the special character @@. In the templates you will find some code preceded with that character, where the data will be inserted by the server.

In the folder Wesby/app/views/, you can find the following HTML templates that you can modify, inside each template you may find some comments to help you with the customization process:

  • main.scala.html

    This is the root template. It defines the header, the styles and the scripts applied to every page in the data portal. The content of the other templates is inserted by the server into the @@content tag. This template is where you can replace the Foundation Framework if you like.

  • resource.scala.html

    It is applied to the resources of the data portal. By default it shows a table listing the properties and its values of the retrieved RDF resource.

  • search.scala.html

    Template for the search results page.

  • welcome.scala.html

    This is the template for this page that you are reading. By default, it is configured as the Index page in the application.conf file, but you can replace it with your own index page once you've finished the configuration process.

  • index.scala.html

    You can use this template to place the content of the index page for your data portal. Once you've finished the configuration you can replace the value of the wesby.index property in application.conf by "/index", so the data portal will display it as the index page instead of this welcome page.
